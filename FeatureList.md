Here is a **clean checklist of features** you should implement while working **only with UI + dummy data**. No API, no state management yet.

---

# HR App UI + Dummy Data Feature Checklist

## 1️⃣ App Layout

* Sidebar navigation
* Top header / navbar
* Main content area
* Responsive layout

---

## 2️⃣ Dashboard

* Total Employees card
* Active Employees card
* Departments count card
* Recent Employees table
* Employees per department chart (optional)

---

## 3️⃣ Employee Module

### Employee Table

* Display employee list
* Search bar
* Column sorting
* Pagination
* Status indicator (Active / Inactive)
* Actions column (Edit / Delete)

### Employee Dialog

* Add Employee form
* Edit Employee form
* Form validation
* Cancel button
* Save button

### Employee Form Fields

* First Name
* Last Name
* Email
* Department
* Role / Job Title
* Salary
* Joining Date
* Active checkbox

---

## 4️⃣ Department Module

### Department Table

* List departments
* Department name
* Manager
* Total employees
* Actions (Edit / Delete)

### Department Dialog

* Add department
* Edit department

---

## 5️⃣ Table Utilities

Apply these to tables:

* Search filter
* Sorting
* Pagination
* Empty state (No data found)
* Loading skeleton (optional)

---

## 6️⃣ UI/UX Enhancements

* Confirmation dialog for delete
* Snackbar / toast message
* Form error messages
* Consistent spacing and typography

---

# Minimum Pages

You should have:

```
Dashboard
Employees
Departments
```
